docker本地项目部署测试

:::info

本地项目部署测试

环境配置：已经pull了python, nginx镜像到本地

1.创建网络
docker network create --subnet=192.167.0.0/16 mydocker

1.1 参数解释：
–subnet=192.167.0.0/16 表示网段IP范围
mydocker表示网络名称

docker network ls

![image.png](https://cdn.nlark.com/yuque/0/2019/png/245407/1567044104502-7a2fbc01-2fb1-4268-9d3a-e3eb7211f024.png)

2.文件挂载和端口映射
docker run -it -v /DRBRAIN:/DRBRAIN -p 8000:8000 --restart=always --net mydocker --ip 192.167.0.2 python:3.6.8 /bin/bash

2.1 参数解释：
run —— 建立容器
-it —— 以交互式运行
-v —— 文件挂载路径，冒号前是宿主机路径，冒号后是docker容器路径
-p —— 端口映射，冒号前是宿主机端口，冒号后是容器开放的端口
–net —— 网络名称
–ip —— IP设置，一般从192.167.0.2开始设置，192.167.0.1为宿主机IP
–restart —— no：容器退出时，不重启容器；
on-failure - 只有在非0状态退出时才从新启动容器；
always - 无论退出状态是如何，都重启容器；

2.2 后续操作:
启动容器
docker start 容器ID
进入容器
docker exec -it 容器ID /bin/bash

3.配置uwsgi.ini

3.1 去宿主机的log目录下新建一个uwsgi.log文件，查看启动状态
cd /DRBRAIN/logs
vi uwsgi.log # 保存退出

3.2 后台启动uwsgi
uwsgi -d --ini uwsgi.ini
查看uwsgi启动状态
ps -aux | grep uwsgi

3.3 日志文件显示以下内容则启动成功

4.配置nginx
4.1创建nginx容器
docker run -it -v /nginx.conf:/etc/nginx/nginx.conf -v /DRBRAIN:/DRBRAIN -p 8080:8080 --net mydocker --ip 192.167.0.3 nginx /bin/bash

4.2 查看nginx状态
service nginx status

4.3启动nginx
nginx

4.4 配置nginx.conf

5.编写DockerFile
5.1 进入dockerdile文件目录
cd /DRBRAIN/Docker/python

5.2 创建dockerfile文件
vi DockerFile

5.3 通过Dockerfile定制镜像（一定要加最后的点）
docker build -t uwsgi .
5.4 保存镜像为tar.gz压缩包上传至服务器
docker save uwsgi -o uwsgi.tar.gz
scp uwsgi.tar.gz root@106.52.24.191:/DRBRAIN
